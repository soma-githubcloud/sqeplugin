# Template Renderer — Reusable Skill

Defines the syntax and rendering rules for `.tmpl` files used by all kit templates.
Load this skill whenever rendering a template to ensure consistent output.

---

## Template Syntax

All templates use double-brace placeholder syntax:

| Syntax | Purpose | Example |
|---|---|---|
| `{{variable}}` | Simple variable substitution | `{{className}}` → `LoginPage` |
| `{{#each items}}...{{/each}}` | Loop over array | Render each locator |
| `{{#if condition}}...{{/if}}` | Conditional block | Include BDD imports |
| `{{#if condition}}...{{else}}...{{/if}}` | If/else block | BDD vs non-BDD spec |
| `{{! comment }}` | Template comment (not output) | `{{! locator section }}` |

---

## Variable Naming Conventions in Templates

| Template Variable | Type | Description |
|---|---|---|
| `{{className}}` | string | PascalCase component/page name |
| `{{projectName}}` | string | camelCase project name |
| `{{kitId}}` | string | e.g., `kit-u1` |
| `{{generatedDate}}` | string | ISO date of generation |
| `{{testStyle}}` | string | `non-bdd` or `bdd` |
| `{{baseUrl}}` | string | Application base URL |
| `{{locators}}` | array | Array of locator objects |
| `{{testCases}}` | array | Array of parsed test case objects |
| `{{imports}}` | array | Import statements to include |
| `{{steps}}` | array | Step objects from test case parser |
| `{{featureName}}` | string | BDD feature name |
| `{{scenarios}}` | array | BDD scenario objects |
| `{{importAlias}}` | string | Import alias for pages (e.g., `@pages` or `../pages`) |
| `{{constructorPattern}}` | string | POM constructor pattern: `"readonly"` or `"standard"` |

## Locator Object Shape (in templates)

```
{{#each locators}}
  alias:    {{this.alias}}       → camelCase alias (e.g., emailInput)
  method:   {{this.method}}      → getByTestId | getByRole | getByLabel | locator
  selector: {{this.selector}}    → the selector value
  name:     {{this.name}}        → role accessible name (optional)
{{/each}}
```

## Rendering Rules

1. **No undefined values in output**: If a template variable is missing from context, render `/* TODO: {{variableName}} */` — never silently skip or output `undefined`

2. **Indentation preservation**: Templates use real indentation (2-space for TS). When rendering loops, maintain the indentation of the `{{#each}}` block for each item.

3. **Line endings**: Always output `\n` (LF). Never `\r\n` even on Windows.

4. **Empty arrays**: If a loop array is empty, render nothing (no empty lines from the loop block).

5. **Conditional BDD imports**: Use `{{#if bdd}}` blocks in config templates to conditionally include Cucumber dependencies.

6. **String escaping**: Values interpolated into string literals must have single quotes escaped: `'` → `\'`

## Generated File Header

Every generated file must start with:

**TypeScript:**
```typescript
// Generated by sqe-kit | Kit: {{kitId}} | {{generatedDate}}
// Source: /sqe-kit:generate-automation-tests
// Do not edit manually — regenerate with: /sqe-kit:generate-automation-tests
```

**Gherkin:**
```gherkin
# Generated by sqe-kit | Kit: {{kitId}} | {{generatedDate}}
# Source test cases: {{testCaseIds}}
```

---

## Post-Render Validation

After rendering any TypeScript template:
- Confirm all `import` paths use relative paths or configured path aliases (`@pages/`, `@fixtures/`)
- Confirm all class names are PascalCase
- Confirm no `any` types appear unless explicitly in a `// TODO: type` context
- Confirm no trailing whitespace on any line
