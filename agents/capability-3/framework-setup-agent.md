# Kit-U1 Framework Setup Agent

**Extends**: `agents/capability-3/scaffolder-agent.md` (resolve via PLUGIN_DIR — read base agent first, then apply U1 specifics below)

**Single Responsibility**: Create a complete, ready-to-run Playwright TypeScript project that supports both non-BDD (`.spec.ts`) and BDD (`.feature` + `.steps.ts`) test styles simultaneously.

> `testStyle = "both"` — both runners are always scaffolded. Do NOT skip either.

---

## Kit-U1 Specific Scaffold Steps

After reading the base scaffolder agent, apply these Kit-U1 specifics.

Read `kit/KIT.md` (resolve via PLUGIN_DIR) for kit-specific generation rules and naming conventions.
Read `kit/kit.config.json` (resolve via PLUGIN_DIR) for `outputDir`, `projectName`, `namingConventions`, and `dependencies`.

### Directory Structure for Kit-U1

All paths below are relative to `<outputDir>/<projectName>/` (resolved from `kit/kit.config.json → outputDir` and `projectName`).

```
<projectPath>/
├── src/
│   ├── pages/                    ← Page Object classes (pomDir)
│   │   └── .gitkeep
│   └── fixtures/                 ← Playwright fixture extensions + data factories (fixturesDir)
│       └── auth.fixtures.ts
├── tests/
│   ├── nonbdd/                   ← Playwright .spec.ts files (testDir)
│   │   └── .gitkeep
│   └── bdd/                      ← Cucumber .feature + .steps.ts files (featureFileDir / stepsFileDir)
│       └── .gitkeep
├── allure-results/
│   └── .gitkeep
├── playwright.config.ts
├── cucumber.json
├── tsconfig.json
├── package.json
└── .gitignore
```

### Playwright Config Specifics (Kit-U1)

`testDir` points to `./tests/nonbdd` only — Playwright must NOT attempt to run `.feature` or `.steps.ts` files.

```typescript
import { defineConfig, devices } from '@playwright/test';
import type { PlaywrightTestConfig } from '@playwright/test';

const config: PlaywrightTestConfig = {
  testDir: './tests/nonbdd',
  fullyParallel: false,
  forbidOnly: !!process.env.CI,
  retries: process.env.CI ? 2 : 0,
  workers: process.env.CI ? 1 : undefined,
  reporter: [
    ['html', { outputFolder: 'playwright-report', open: 'never' }],
    ['allure-playwright', { detail: true, outputFolder: 'allure-results' }],
    ['list']
  ],
  use: {
    baseURL: process.env.BASE_URL || 'http://localhost:3000',
    screenshot: 'on',
    video: 'on',
    trace: 'on-first-retry',
  },
  projects: [
    { name: 'chromium', use: { ...devices['Desktop Chrome'] } }
  ]
};

export default config;
```

### Cucumber Config (cucumber.json) — Always Generated

Cucumber reads from `tests/bdd/`. Step files use the `.steps.ts` suffix defined in `kit/kit.config.json → namingConventions.stepFileSuffix`.

### Post-install verification (Kit-U1 specific)

After `npm install`, run all three checks in order. All must pass before reporting completion to the user.

| Step | Command | Expected result |
|---|---|---|
| 1 | `npx playwright install chromium` | Chromium downloaded, exit 0 |
| 2 | `npx tsc --noEmit` | 0 TypeScript errors |
| 3 | `npx playwright test --list` | 0 tests found, no errors (no spec files yet) |
| 4 | `npx cucumber-js --dry-run` | 0 scenarios found, no errors (no feature files yet) |

If any step fails, diagnose and fix before proceeding.
